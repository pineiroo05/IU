Informacion sobre las clases de test

Clase TestForm

Descripcion: clase encargada de evaluar de forma dinámica las estructuras y definiciones de test definidos para cada entidad, y de ejecutar las pruebas.

Metodo: ejecutar(nombreEntidad)

Descripcion:

ParametroTipoDescripcion
nombreEntidadStringNombre de la entidad seleccionada

Informacion devuelta: void

Clase Gestor

Descripcion: clase encargada de controlar la interfaz. Se encarga de controlar que las variables de cada entidad existen y en caso de que existan se muestran los botones de test. En caso negativo se encarga de crear el modal de error correspondiente.

Metodo: atenderMenu(nombreEntidad)

Descripcion: metodo que responde a la seleccion de una entidad concreta, preparando la interfaz para mostrar los test.

ParametroTipoDescripcion
nombreEntidadStringNombre de la entidad seleccionada

Informacion devuelta: void

Metodo: comprobarVariables(nombreEntidad)

Descripcion: metodo encargado de comprobar que todas las variables especificadas en la estructura correspondiente a la entidad seleccionada existen.

ParametroTipoDescripcion
nombreEntidadStringNombre de la entidad seleccionada

Informacion devuelta: void

Metodo: mostrarPanel(nombreEntidad)

Descripcion: método encargado de mostrar la interfaz con los botones para los test de atributos y formulario.

ParametroTipoDescripcion
nombreEntidadStringNombre de la entidad seleccionada

Informacion devuelta: void

Metodo: mostrarError(mensaje)

Descripcion: metodo que interrumpe la ejecucion normal, para poder mostrar el modal de error en caso de que falto algo de la estructura

ParametroTipoDescripcion
mensajeStringMensaje a mostrar en caso de error

Informacion devuelta: void

Metodo: cerrarModal()

Descripcion: oculta los modales activos en la pantalla.

Informacion devuelta: void

Información sobre las clases de validación

Clase: ValidateFieldsForm

Descripción:clase encargada de gestionar el entorno de pruebas. Genera de forma dinamica elementos temporales dentro de un contenedor, le inyecta los valores de pruebas y ejecuta las pruebas correspondientes.

Método: crearCampo(id, valor, infoAtributo)

Descripción: Crea un elemento HTML según las normas de la estructura de la entidad, configura sus propiedades, le asigna el valor de prueba y lo añade al DOM.

ParámetroTipoDescripción
idStringIdentificador del nuevo nodo HTML.
valorString/ObjectDato de prueba a introducir en el campo (puede ser un objeto para simular ficheros).
infoAtributoObjectEstructura que contiene el html.

Información devuelta: HTMLElement-El nodo del elemento de formulario recién creado y colocado en el DOM.

Método: validarCampo(validacion, id, parametros)

Descripción: Invoca al metodo de validacion concreto de cada campo.

ParámetroTipoDescripción
validacionStringNombre del método a ejecutar.
idStringID del elemento HTML temporal que contiene el valor que se va a evaluar.
parametrosMixedCriterios de validacion de la regla.

Información devuelta: Boolean/String-Devuelve true si pasa la prueba. Si falla, devuelve el codigo de error. Si la regla existe, devuelve un mensaje de aviso.

Método: simulacionFichero(campo, objeto)

Descripción: crea un fichero simulado para el testeo de los campos relacionados con los ficheros.

ParámetroTipoDescripción
campoHTMLElementEl nodo input file donde se monta el archivo.
objetoObjectElemento donde se describen las caracteristicas del fichero a crear (tamaño,tipo,nombre...).

Información devuelta:void

Método: limpiar()

Descripción: Limipia por completo el contenedor temporal eliminando todos los elementos HTML que se añadieron previamente.

Parámetros: Ninguno.

Información devuelta:void

Clase: project

Descripcion: esta clase contiene los metodos de validacion personalizados para la entidad "project"

Metodo: start_date_project_personalizaed_validation()

Descripcion: comprueba que la fecha de inicio no sea mayor que la fecha de finalizacion del proyecto.

Parámetros: Ninguno.

Informacion devuelta: boolean/string, en el caso en el que la fecha de inicio sea mayor a la da de fin devuelve el codigo de error, en el caso en el que sea mayor, o no haya introducida una fecha (caso especial de las busquedas) devuelve true.

Metodo: end_date_project_personalized_validation()

Descripcion: comprueba que la fecha de final de proyecto no sea menor que la de inicio.

Parámetros: Ninguno.

Informacion devuelta: boolean/string, en el caso en el que la fecha de fin sea menor a la de inicio devuelve el codigo de error, en caso de que sea mayor o no haya introducida una fecha (caso especial de las busquedas) devuelve true.

Clase: characteristic

Descripcion: esta clase contiene los metodos de validacion personalizados para la entidad "characteristic"

Metodo: data_type_characteristic_personalized_validation(accion)

Descripcion: comprueba que el valor seleccionado para el campo data_type de la entidad sea uno de los valores aceptados (number, text o set).

ParametrosTipoDescripcion
accionstringIdentifica la accion que se esta llevando a cabo (ADD, EDIT o SEARCH)

Informacion devuelta: boolean/string, true en caso de que el valor introducido sea valido o bien si el campo esta vacio (solo si es una busqueda). En caso de que el valor no sea valido, devuelve el codigo de error.

Metodo: category_characteristic_personalized_validation(accion)

Descripcion: comprueba que el valor seleccionado para el campo category de la entidad sea uno de los valores aceptados (soil_site, soil_chem o soil_bio).

ParametrosTipoDescripcion
accionstringIdentifica la accion que se esta llevando a cabo (ADD, EDIT o SEARCH)

Informacion devuelta: boolean/string, true en caso de que el valor introducido sea valido o bien si el campo esta vacio (solo si es una busqueda). En caso de que el valor no sea valido, devuelve el codigo de error.

Informacion sobre las clases de test

Clase TestForm

Descripcion: clase encargada de evaluar de forma dinámica las estructuras y definiciones de test definidos para cada entidad, y de ejecutar las pruebas de atributos.

Metodo: ejecutar(nombreEntidad)

Descripcion: metodo encargado de comprobar la existencia de los ficheros asociados a cada entidad y de ejecutar todas las pruebas. Crea los objetos que luego se insertan en el modal correspondiente.

Parametros:

ParametroTipoDescripcion
nombreEntidadStringNombre de la entidad seleccionada

Informacion devuelta: void

Metodo: cargarDatosEntidad(nombreEntidad)

Descripcion: intenta cargar de forma dinamica utilizando eval, los js de los datos asociados a la entidad.

Parametros:

ParametroTipoDescripcion
nombreEntidadstringNombre de la entidad de la cual queremos cargar los datos.

Informacion devuelta: Object, contiene toda la informacion asociada a la entidad.

Metodo: procesarEstructura(estructura,nombreEntidad)

Descripcion: procesa el objeto que contiene la informacion de la entidad (obtenido en cargarDatosEntidad), para poder generar el bloque de resultados. Comprueba si la estructura existe y lista sus atributos.

Parametros:

ParametroTipoDescripcion
estructuraObjectObjeto que contiene el Json de la estructura considerada.

Informacion devuelta: String, html correspondiente al primer bloque a mostrar.

Metodo: procesarDefinicionTests(defTests,nombreEntidad)

Descripcion: evalua el formato del array de definiciones de pruebas de atributos. Realiza el control acumulando el conteo de las pruebas planificadas por cada campo.

Parametros:

ParametroTipoDescripcion
defTestsArrayArray con las configuraciones de tests de atributos.

Informacion devuelta: String, html correspondiente a la informacion del segundo bloque.

Metodo: procesarPruebas(datos, nombreEntidad)

Descripcion: recorre el conjunto de test de atributos. Crea los campos en el validador, ejecuta las reglas y compara el resultado obtenido con el esperado.

Parametros:

ParametroTipoDescripcion
datosObjectEstructura completa de datos obtenida en la carga dinámica

Informacion devuelta: Object. Contiene un html, y el array de resultados.

Metodo: ejecutarReglasValidaciones(campoNombre, accion, reglas, entidad)

Descripcion: método que aplica de forma secuencial y en orden las reglas de tamaño, formato o validaciones personalizadas del atributo.

Parametros:

ParametroTipoDescripcion
campoNombreStringNombre del atributo bajo análisis
accionStringAcción a realizar (SEARCH, ADD, EDIT...)
reglasObjectConjunto de resticciones aplicables.
entidadObjectInstancia de la clase de la entidad para invocar funciones personalizadas

Informacion devuelta: Boolean/String, true si supera los filtros, en caso contrario devuelve el codigo de error.

Metodo: configurarBotonDetalles(resultados, nombreEntidad)

Descripcion: metodo que inyecta el evento dinámico (onclick) al botón encargado de desplegar la ventana modal del sistema, pintando las filas en funcion de si la prueba fue un éxito o un fallo

Parametros:

ParametroTipoDescripcion
resultadosArrayColección de objetos procesados con el estado final de los tests ejecutados
nombreEntidadStringNombre de la entidad seleccionada

Informacion devuelta: void

Clase Gestor

Descripcion: clase encargada de controlar la interfaz. Se encarga de controlar que las variables de cada entidad existen y en caso de que existan se muestran los botones de test. En caso negativo se encarga de crear el modal de error correspondiente.

Metodo: atenderMenu(nombreEntidad)

Descripcion: metodo que responde a la seleccion de una entidad concreta, preparando la interfaz para mostrar los test.

Parametros:

ParametroTipoDescripcion
nombreEntidadStringNombre de la entidad seleccionada

Informacion devuelta: void

Metodo: comprobarVariables(nombreEntidad)

Descripcion: metodo encargado de comprobar que todas las variables especificadas en la estructura correspondiente a la entidad seleccionada existen.

Parametros:

ParametroTipoDescripcion
nombreEntidadStringNombre de la entidad seleccionada

Informacion devuelta: void

Metodo: mostrarPanel(nombreEntidad)

Descripcion: método encargado de mostrar la interfaz con los botones para los test de atributos y formulario.

Parametros:

ParametroTipoDescripcion
nombreEntidadStringNombre de la entidad seleccionada

Informacion devuelta: void

Metodo: mostrarError(mensaje)

Descripcion: metodo que interrumpe la ejecucion normal, para poder mostrar el modal de error en caso de que falto algo de la estructura

Parametros:

ParametroTipoDescripcion
mensajeStringMensaje a mostrar en caso de error

Informacion devuelta: void

Metodo: cerrarModal()

Descripcion: oculta los modales activos en la pantalla.

Parametros: ninguno

Informacion devuelta: void

Información sobre las clases de validación

Clase: ValidateFieldsForm

Descripción:clase encargada de gestionar el entorno de pruebas. Genera de forma dinamica elementos temporales dentro de un contenedor, le inyecta los valores de pruebas y ejecuta las pruebas correspondientes.

Método: crearCampo(id, valor, infoAtributo)

Descripción: Crea un elemento HTML según las normas de la estructura de la entidad, configura sus propiedades, le asigna el valor de prueba y lo añade al DOM.

Parametros:

ParámetroTipoDescripción
idStringIdentificador del nuevo nodo HTML.
valorString/ObjectDato de prueba a introducir en el campo (puede ser un objeto para simular ficheros).
infoAtributoObjectEstructura que contiene el html.

Información devuelta: HTMLElement-El nodo del elemento de formulario recién creado y colocado en el DOM.

Método: validarCampo(validacion, id, parametros)

Descripción: Invoca al metodo de validacion concreto de cada campo.

Parametros:

ParámetroTipoDescripción
validacionStringNombre del método a ejecutar.
idStringID del elemento HTML temporal que contiene el valor que se va a evaluar.
parametrosMixedCriterios de validacion de la regla.

Información devuelta: Boolean/String-Devuelve true si pasa la prueba. Si falla, devuelve el codigo de error. Si la regla existe, devuelve un mensaje de aviso.

Método: simulacionFichero(campo, objeto)

Descripción: crea un fichero simulado para el testeo de los campos relacionados con los ficheros.

Parametros:

ParámetroTipoDescripción
campoHTMLElementEl nodo input file donde se monta el archivo.
objetoObjectElemento donde se describen las caracteristicas del fichero a crear (tamaño,tipo,nombre...).

Información devuelta:void

Método: limpiar()

Descripción: Limipia por completo el contenedor temporal eliminando todos los elementos HTML que se añadieron previamente.

Parámetros: Ninguno.

Información devuelta:void

Clase: project

Descripcion: esta clase contiene los metodos de validacion personalizados para la entidad "project"

Metodo: start_date_project_personalizaed_validation()

Descripcion: comprueba que la fecha de inicio no sea mayor que la fecha de finalizacion del proyecto.

Parámetros: Ninguno.

Informacion devuelta: boolean/string, en el caso en el que la fecha de inicio sea mayor a la da de fin devuelve el codigo de error, en el caso en el que sea mayor, o no haya introducida una fecha (caso especial de las busquedas) devuelve true.

Metodo: end_date_project_personalized_validation()

Descripcion: comprueba que la fecha de final de proyecto no sea menor que la de inicio.

Parámetros: Ninguno.

Informacion devuelta: boolean/string, en el caso en el que la fecha de fin sea menor a la de inicio devuelve el codigo de error, en caso de que sea mayor o no haya introducida una fecha (caso especial de las busquedas) devuelve true.

Clase: characteristic

Descripcion: esta clase contiene los metodos de validacion personalizados para la entidad "characteristic"

Metodo: data_type_characteristic_personalized_validation(accion)

Descripcion: comprueba que el valor seleccionado para el campo data_type de la entidad sea uno de los valores aceptados (number, text o set).

Parametros:

ParametrosTipoDescripcion

Informacion devuelta: boolean/string, true en caso de que el valor introducido sea valido o bien si el campo esta vacio (solo si es una busqueda). En caso de que el valor no sea valido, devuelve el codigo de error.

Metodo: category_characteristic_personalized_validation(accion)

Descripcion: comprueba que el valor seleccionado para el campo category de la entidad sea uno de los valores aceptados (soil_site, soil_chem o soil_bio).

Parametros:

ParametrosTipoDescripcion
accionstringIdentifica la accion que se esta llevando a cabo (ADD, EDIT o SEARCH)

Informacion devuelta: boolean/string, true en caso de que el valor introducido sea valido o bien si el campo esta vacio (solo si es una busqueda). En caso de que el valor no sea valido, devuelve el codigo de error.

Informacion sobre las clases de submit

Clase TestSubmit

Descripcion: Clase encargada de ejecutar la validacion simulando el envío de un formulario completo. Evalúa las reglas de todos los atributos de una entidad, recopilando los errores detectados.

Metodo: checkSubmit(nombreEntidad, entidad, accion, datos, estructura)

Descripcion: Simula el envio del formulario cargando todos los atributos definidos en la estructura. Itera sobre las restricciones de cada campo bajo un orden estricto de prioridades y recopila en un array los mensajes de error únicos generados.

Parametros:

ParametroTipoDescripcion
nombreEntidadStringNombre que identifica a la entidad a procesar.
entidadObjectInstancia que contiene los métodos dinámicos de validación personalizados.
accionStringAccion a realizar.
datosObjectContiene los pares clave-valor que simulan los datos introducidos en el formulario.
estructuraObjectObjeto que contiene la estructura de los atributos y restricciones de validación de la entidad.

Informacion devuelta: Boolean/Array. Retorna true si el formulario es completamente válido, o un Array con los errores unicos encontrados.

Metodo: ejecutar(nombreEntidad)

Descripcion: comprueba que los datos y estructuras necesarios existan, e invoca el procesamiento las pruebas, inyecta la tabla de resumen por acciones en el DOM y activa el cuadro de detalles.

Parametros:

ParametroTipoDescripcion
nombreEntidadStringEntidad sobre la que se hacen las pruebas.

Informacion devuelta: void

Metodo: cargaDatosEntidad(nombreEntidad)

Descripcion: Recupera toda la informacion asociada a la entidad: su estructura, pruebas de submit, y crea una instancia de la clase de la entidad para acceder a las pruebas personalizadas.

Parametros:

ParametroTipoDescripcion
nombreEntidadStringEntidad de la que se quiere cargar su informacion.

Informacion devuelta: Object. Contiene las caracteristicas de cada entidad (pruebas, estructura, entidad).

Metodo: procesarPruebasSubmit(datos, nombreEntidad)

Descripcion: Recorre el array completo de pruebas de submit. Ejecuta la función de control para cada caso y compara los errores esperados respecto a los obtenidos, y almacena el numero de pruebas correctas y erroneas de cada accion.

Parametros:

ParametroTipoDescripcion
datosObjectObjeto que contiene las pruebas, entidad y estructura de datos cargadas.
nombreEntidadStringNombre de la entidad seleccionada.

Informacion devuelta: Object. Devuelve un objeto estructurado con las propiedades: resumen [contadores estadísticos agrupados] y resultados [matriz detallada de mapeos individuales para el modal]).

Metodo: procesarResumenAcciones(resumen)

Descripcion: Construye la tabla resumen de las pruebas realizadas, aciertos y fallos por cada acción.

Parametros:

ParametroTipoDescripcion
resumenObjectEstructura con el recuento total, de pruebas correctas e incorrectas de las acciones evaluadas.

Informacion devuelta: String. Código HTML de la tabla resumen.

Metodo: configurarBotonDetalles(resultados, nombreEntidad)

Descripcion: metodo que inyecta el evento dinámico (onclick) al botón encargado de desplegar la ventana modal del sistema, pintando las filas en funcion de si la prueba fue un éxito o un fallo.

Parametros:

ParametroTipoDescripcion
resultadosArrayArray que contiene el estado de cada test realizado.
nombreEntidadStringNombre de la entidad seleccionada.

Informacion devuelta: void
